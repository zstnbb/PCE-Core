## Three-Phase Test Design

- **Phase 1 (Connectivity):** Login check + input element detection
- **Phase 2 (Capture + Storage):** Send message → wait for response → verify captures in DB with correct provider/session/messages
- **Phase 3 (Render):** Verify dashboard API returns renderable session data

## Recent Fixes (This Session)

### Kimi E2E — 3 Root Causes Fixed

1. **Provider misattribution (`generic.js`)**
   - `HOST_PROVIDER_MAP` was missing `kimi.com` / `www.kimi.com` / `kimi.moonshot.cn`
   - Fallback `hostname.split(".")[0]` produced `"www"` instead of `"moonshot"`
   - Fix: Added all three Kimi domains to the map

2. **DOM message extraction failure (`generic.js`)**
   - Kimi uses a segment-based chat layout: `.segment.segment-user` / `.segment.segment-assistant`
   - None of the existing generic selectors matched Kimi's user messages
   - Fix: Added `.segment.segment-user, .segment.segment-assistant` as first selector

3. **`ERR_CONNECTION_RESET` on navigation (`kimi.py`)**
   - Repeated `driver.get("https://kimi.com/...")` caused Selenium `net::ERR_CONNECTION_RESET`
   - Fix: `KimiAdapter` overrides `check_logged_in` and `navigate` to:
     - Skip `driver.get()` if already on a Kimi page
     - Use `execute_script("window.location.href = ...")` for softer JS navigation

4. **Normalizer host registration**
   - Added `www.kimi.com` and `kimi.com` to `_COMPATIBLE_HOSTS` (openai.py) and `_CONVERSATION_HOSTS` (conversation.py)

### Previous Session Fixes

- **Clipboard paste upload** — Universal `upload_via_paste` method in `BaseSiteAdapter` using JS `ClipboardEvent` + `DataTransfer`. Enables file/image upload on sites without `input[type="file"]` (Gemini, Manus, Kimi).
- **Rich content capture universalization** — Shared `pce_dom_utils.js`, noise filtering, session normalization, dedup improvements.
