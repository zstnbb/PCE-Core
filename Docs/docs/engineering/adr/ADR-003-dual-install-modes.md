# ADR-003: 采用轻量模式与完整模式双轨安装策略

- Status: Accepted
- Date: 2026-04-05

## Context

如果只提供系统级完整方案，虽然覆盖最彻底，但会要求用户安装本地证书、接受系统级代理配置，信任门槛较高。

如果只提供轻量方案，门槛较低，但覆盖范围不足，产品形态又会变得碎片化。

用户对这两种代价的承受能力并不一致。

## Decision

PCE 采用双轨模式：

### Light Mode
- 通过环境变量、局部 API 路由、浏览器插件等方式采集数据
- 目标是快速上手与降低信任门槛

### Complete Mode
- 通过系统级 HTTPS 代理和本地 CA 证书尽量统一覆盖 AI 域名流量
- 目标是最大覆盖率与未来注入能力

两种模式共用同一后端：

- 相同数据库
- 相同查询层
- 相同 dashboard
- 相同未来分析能力入口

## Rationale

1. 允许用户按自己的承受能力选择接入深度。
2. 在不放弃“最彻底形态”的前提下，保留更安全、更稳妥的验证入口。
3. 可以先用轻量模式验证价值，再升级到完整模式。
4. 产品不被单一安装模式绑死。

## Consequences

### Positive
- 用户选择空间更大
- 降低早期采用阻力
- 技术路线更灵活

### Negative
- 需要维护两套接入入口
- 需要在 UI 与文档中明确说明差异
- 某些功能在两种模式下可用性可能不同

## Follow-up

后续安装向导必须明确展示：

- 两种模式的覆盖范围
- 两种模式的风险与授权要求
- 模式可否切换与升级
- 当前运行模式下的数据来源状态
