## Summary

Second round of site-extractor migration. Ports `deepseek.js`,
`google_ai_studio.js`, and `perplexity.js` to TypeScript using the
`utils/capture-runtime.ts` factory delivered in batch 1. No new runtime
features — this is a strict behaviour-parity port of three extractors
using the exact pattern established for ChatGPT / Claude / Gemini.

After this batch, **6 of 13 site extractors** are on TypeScript. The
remaining 7 (copilot, poe, grok, huggingface, manus, zhipu, generic +
universal_extractor) will land across batches 3-5.

## Deliverables

### 1. `entrypoints/deepseek.content.ts` (~330 lines)

**Matches**: `https://chat.deepseek.com/*`

| Helper | Purpose |
|---|---|
| `getSessionHint(pathname)` | `/chat/<id>` or `/chat/s/<id>` → id; pathname fallback |
| `getContainer(doc)` | `main` else body |
| `findTurnContainer(el)` | Walk ≤ 6 levels up from `.ds-markdown` to find a parent with user-side siblings |
| `extractUserFromTurn(turn, md)` | Return the sibling text that does NOT contain `.ds-markdown` |
| `cleanText(el)` | Strip icons/buttons/avatars/toolbars; prefer inner `.ds-markdown` |
| `extractMessages(doc)` | 3 strategies: `.ds-markdown` anchor → `[data-role]` → large-text-block heuristic |
| `getModelName(doc)` | 7-selector ladder → title regex → body-text (`DeepSeek-R1` / `DeepSeek-V3`) → `DeepSeek` fallback |

**Capture mode**: `incremental`
**SPA nav**: `hookHistoryApi: false` (URL polling only)
**Manual capture bridge**: wired via `pce-manual-capture` listener
**Debounce / poll**: 2500 ms / 5000 ms (matches legacy)

### 2. `entrypoints/google-ai-studio.content.ts` (~500 lines)

**Matches**: `https://aistudio.google.com/*`

Largest port in batch 2 — the legacy file was 469 lines with extensive
site-specific logic. Every helper is exported at module scope so the
Vitest suite can exercise the extraction ladder independently of the
`defineContentScript` closure.

| Helper | Purpose |
|---|---|
| `normalizeText(text)` | Strip `CONTROL_LINE_RE` (edit/more_vert/thumb_up/…), `META_LINE_RE` (`user 12:34`), `DISCLAIMER_LINE_RE`, `AI_STUDIO_UI_LINE_RE`; collapse internal whitespace + dedupe consecutive duplicate lines |
| `dedupeAttachments(list)` | Composite-key dedup on type/url/name/title/code |
| `imageMediaType(src, name)` | Parse `data:` URL header or infer from extension (png/jpg/jpeg/gif/webp) |
| `attachmentOnlyText(atts)` | `[Image attachment: …]` / `[File attachment: …]` labels for attachment-only messages |
| `extractLocalAttachments(el, origin)` | Extract `<ms-image-chunk>` images, `<ms-file-chunk>` files, `<pre>` code blocks, external `<a href>` citations (skips same-origin) |
| `cleanContainerText(el, {stripCode, stripLinks})` | Strip AI-Studio chrome (buttons/mat-icons/feedback/author-label/timestamp/turn-footer), optionally drop code/links |
| `isStreaming(doc)` | Shared DOM helper OR Stop/Cancel button detection |
| `getContainer(doc)` | `.chat-view-container` / `ms-chat-session` / `.chat-session-content` / `main` / body |
| `getSessionHint(pathname)` | `/prompts/<id>` → id; pathname fallback |
| `getModelName(doc)` | 5-selector ladder → body-text `gemini-*` regex |
| `extractUserText(turn)` | `cleanContainerText(user container)` |
| `extractAssistantText(turn, atts)` | Adaptive strip based on attachment types; reply + thinking wrap |
| `extractMessages(doc, origin)` | Iterate `ms-chat-turn` → classify user/model/ambiguous |

**Capture mode**: `incremental`
**SPA nav**: `hookHistoryApi: false` (URL polling only)
**Manual capture bridge**: wired via `pce-manual-capture` listener
**Debounce / stream-retry / poll**: 2500 / 1500 / 4000 ms
**Streaming detection**: actively defers capture while a Stop/Cancel
generation button is visible in the DOM.

## Design Decisions

### Why port the three extractors together?

They share the same capture strategy — incremental delta, URL-polling
SPA nav, `pce-manual-capture` bridge — and all three fit cleanly into
the capture-runtime pattern. Porting them in one PR lets the reviewer
see the third extractor land without the shared runtime needing any
changes, confirming that batch 1's factory abstraction is stable.

### Why didn't AI Studio's `normalizeText` move to `utils/pce-dom.ts`?

The regex tables (`CONTROL_LINE_RE`, `META_LINE_RE`,
`DISCLAIMER_LINE_RE`, `AI_STUDIO_UI_LINE_RE`) are AI-Studio-specific
UI conventions. Other sites strip different junk. Promoting
`normalizeText` to the shared utils would invite other extractors to
co-opt it with their own regex noise, producing a creeping
kitchen-sink helper. The text normalizer stays local to the site
extractor.

### Why no `history.pushState` hook for any batch 2 site?

Legacy parity. None of the three legacy JS files intercepted the
history API — they all relied purely on URL polling. Adding
`hookHistoryApi` now would be a behaviour change, not a port. If
future e2e evidence shows missed SPA navigations on these sites, the
hook can be flipped on in a separate PR.

### Why doesn't Perplexity report a model name?

Legacy parity: `perplexity.js` always sent `model_name: null`. The TS
port mirrors that by passing `getModelName: () => null`. Perplexity's
UI doesn't expose a stable "model" label because it mixes retrieval +
LLM into a single "answer" abstraction.

### Why does AI Studio get a streaming hook but the others don't?

Again, legacy parity — `google_ai_studio.js` was the only one of the
three that actively deferred captures while a Stop/Cancel generation
button was visible. The others started captures as soon as the DOM
mutated; the fingerprint diff effectively dedup'd partial-streaming
noise for them.

### Why are the three extractors' local extractors allowed to reach into `utils/pce-dom.ts` functions like `normalizeCitationUrl`?

Those helpers are generic and already unit-tested in Phase 3a. Re-use
is cheaper than duplication. Only site-specific logic stays in the
entrypoint file.

## Risks mitigated

- **Site-specific regressions**: 50 focused Vitest cases enumerate the
  extraction heuristics for each site. Every code path in the legacy
  JS has a corresponding TS test.
- **AI Studio noise regressions**: Each of the 4 noise regex classes
  (control lines, meta lines, disclaimer, UI actions) has a dedicated
  test. The attachment dedup key format is asserted explicitly.
- **DeepSeek DOM walk-up**: The `findTurnContainer` 6-level cap and
  the "sibling has user text" heuristic are covered. The not-found
  path is covered to prevent regressions where a degenerate DOM
  returns phantom containers.
- **Legacy JS drift**: The legacy `.js` files under
  `../pce_browser_extension/content_scripts/` remain unchanged. The
  reduced-bundle entry still injects the site-independent helpers, so
  `__PCE_BEHAVIOR` + the Save-snippet button keep working on the
  ported sites.
