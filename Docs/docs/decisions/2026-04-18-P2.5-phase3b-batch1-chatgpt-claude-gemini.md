## Summary

Phase 3b batch 1 is the first of five site-extractor migration rounds. It
takes the three highest-traffic extractors — `chatgpt.js`, `claude.js`,
`gemini.js` — off the legacy JS directory and ships them as TypeScript
`defineContentScript` entrypoints. A new `utils/capture-runtime.ts`
factors out the ~80 % boilerplate these three shared (MutationObserver
lifecycle, fingerprint diffing, debounced capture, SPA nav, rollback
semantics) so subsequent batches can follow the same pattern with much
less duplication.

Behaviour parity is strict: every timing constant, fingerprint format,
message-delta semantic, DOM selector, and error-rollback flow was lifted
byte-for-byte from the legacy JS. No functional changes; all Python
backend tests remain green.

## Deliverables

### 1. New shared capture runtime — `utils/capture-runtime.ts`

```
export function createCaptureRuntime(options: CaptureRuntimeOptions): CaptureRuntime
```

Site extractors instantiate this factory with their site-specific
callbacks (`getContainer`, `extractMessages`, `getSessionHint`,
`getModelName`, `isStreaming`, `resolveConversationId`) and timing
constants, then call `runtime.start()` at `DOMContentLoaded`.

The runtime owns:

| Concern | Implementation |
|---|---|
| Container discovery + retry | Up to 15 × 2 s attempts before giving up; resets counter on success |
| Observer lifecycle | `MutationObserver` on container with `{ childList, subtree, characterData }` |
| Debounce | `setTimeout(captureNow, debounceMs)` coalescing |
| Streaming deferral | If `isStreaming()` returns true, reschedule after `streamCheckMs` |
| Fingerprint diff | `fingerprintConversation()` from `pce-dom.ts`; skip send when unchanged |
| Capture modes | `incremental` (ChatGPT/Gemini) — sends `sentCount`-based delta + `message_update` when last message extended; `full` (Claude) — always resends all |
| Conversation ID | Optional `resolveConversationId(messages)` override (ChatGPT's `_new_` + timestamp pattern); falls back to `getSessionHint()` |
| SPA nav — history API | Optional `hookHistoryApi: true` patches `pushState` / `replaceState` + listens to `popstate` |
| SPA nav — URL polling | Always installed; polls `location.href` every `pollIntervalMs` |
| Nav reset | Clears `sentFingerprint` + `sentCount` + fires optional `onNavChange` callback |
| Rollback | Reverts fingerprint + sentCount on `chrome.runtime.lastError` or non-`ok` response |
| Behaviour snapshot | Reads `window.__PCE_BEHAVIOR?.getBehaviorSnapshot(true)` at payload assembly, `{}` fallback |

All I/O is injectable — `win`, `doc`, `chromeRuntime`, `logger` default
to real globals but tests supply stubs so the runtime runs under
happy-dom without touching real extension APIs.

### 2. Site-specific entrypoints (all TypeScript)

#### `entrypoints/chatgpt.content.ts` (~370 lines, 22 exported helpers)

**Matches**: `https://chatgpt.com/*`, `https://chat.openai.com/*`

| Helper | Purpose |
|---|---|
| `getConvId(pathname)` | `/c/<uuid>` → uuid; null otherwise |
| `getContainer(doc)` | 5-selector priority list ending at `main` |
| `isStreamingChatGPT(doc)` | Stop button + `result-streaming` + `agent-turn .streaming` |
| `extractSpecialContent(doc)` | Deep Research / Canvas fallback for assistant-only scenario |
| `extractMessages(doc)` | 5 strategies: `[data-message-author-role]` → `[data-testid^=conversation-turn]` → `article` → `[data-message-id]` → ARIA rows |
| `getModelName(doc)` | First `[class*=model]` or `[data-testid*=model]` text |
| `resolveConversationId(messages, pathname, now)` | URL UUID → else `_new_` + timestamp when ≥ 2 msgs → else null |

**Capture mode**: `incremental`
**SPA nav**: `hookHistoryApi: true` (pushState + popstate)
**Manual capture bridge**: `installManualCaptureBridge(document)` +
`pce-manual-capture` listener that calls `runtime.triggerCapture()`.

#### `entrypoints/claude.content.ts` (~240 lines, 3 exported helpers)

**Matches**: `https://claude.ai/*`

| Helper | Purpose |
|---|---|
| `getSessionHint(pathname)` | `/chat/<uuid>` → uuid |
| `getContainer(doc)` | 6-selector priority list ending at `main` |
| `extractMessages(doc)` | 3 strategies: dedicated human/assistant selectors sorted by visual top → generic `[class*=message]` with class/data-role/aria inference → direct-children-of-container alternating |

**Capture mode**: `full` (legacy Claude always resent the whole
conversation, not a delta).
**SPA nav**: `hookHistoryApi: false` (URL polling only — matches
legacy).
**Manual capture bridge**: NOT wired (legacy omission preserved).

#### `entrypoints/gemini.content.ts` (~220 lines, 6 exported helpers)

**Matches**: `https://gemini.google.com/*`

| Helper | Purpose |
|---|---|
| `getSessionHint(pathname)` | `/app/<hex>` or `/chat/<hex>` → id; else pathname |
| `getContainer(doc)` | `main` else body |
| `getModelName(doc)` | `.model-selector .selected` / `[data-model-name]` / `.model-badge` |
| `detectRole(el)` | Tag (`user-query` / `model-response`) → `data-turn-role` → class keywords |
| `extractText(el)` | `extractReplyContent` for model-response; noise-strip + markdown-child for rest |
| `extractMessages(doc)` | 4 strategies with dedup, then `[class*=turn]` fallback |

**Capture mode**: `incremental`
**SPA nav**: `hookHistoryApi: false` (URL polling only).
**Manual capture bridge**: wired via `pce-manual-capture` listener.

### 3. Manifest surgery — `wxt.config.ts`

Split the legacy `SITE_SCRIPT_COMMON` into two groups:

```ts
const SITE_INDEPENDENT_HELPERS = [
  "content_scripts/behavior_tracker.js",
  "content_scripts/text_collector.js",
  "content_scripts/detector.js",
] as const;

const LEGACY_EXTRACTOR_DEPS = [
  "content_scripts/pce_dom_utils.js",
  "content_scripts/site_configs.js",
  "content_scripts/selector_engine.js",
] as const;

const LEGACY_SITE_SCRIPT_COMMON = [
  ...LEGACY_EXTRACTOR_DEPS,
  ...SITE_INDEPENDENT_HELPERS,
] as const;
```

- **Ported sites** (ChatGPT / Claude / Gemini) get ONLY
  `SITE_INDEPENDENT_HELPERS` via a single shared content-script entry.
  The TS `defineContentScript` entrypoints handle the rest.
- **Legacy sites** (10 remaining) keep the full
  `LEGACY_SITE_SCRIPT_COMMON` + their `<site>.js` via the renamed
  `legacySiteBundle(site, extractor)` helper.
- The imperative `content_scripts` list no longer mentions `chatgpt.js`,
  `claude.js`, or `gemini.js`.

The legacy files still exist under `../pce_browser_extension/content_scripts/`
and are still copied into `public/content_scripts/` by
`scripts/sync-legacy-assets.mjs` — that's harmless dead weight that
Phase 4 will clean up.

## Design Decisions

### Why extract a shared runtime instead of copying the boilerplate?

The three extractors had near-identical
MutationObserver + debounce + fingerprint + sendMessage + rollback +
SPA-nav plumbing. Without factoring, each new TS extractor would
duplicate ~200 lines of subtle stateful logic, and fixes to the
rollback semantics (for instance) would need to be applied in 13
places by Phase 3b completion. The runtime is injectable (no globals)
and every behaviour is covered by a test, so regressions in
subsequent batches surface as runtime-test failures rather than
site-specific bugs discovered in the field.

### Why are ported sites still getting `behavior_tracker.js` + `text_collector.js`?

`behavior_tracker.js` writes `window.__PCE_BEHAVIOR` which the capture
runtime reads when assembling the `meta.behavior` payload field. The
legacy extractor also read this global; we preserve that behaviour.
`text_collector.js` is the floating "Save snippet" button — entirely
independent of the extractor but part of the user-visible capture UX.
Both will migrate to TS in Phase 3b batch 5.

### Why is `claude.content.ts` in `full` mode while the others are `incremental`?

Strict parity with `claude.js`. The legacy Claude script always
resent the entire conversation when the fingerprint changed. The
reason (inferred) is that Claude's turn DOM is more volatile than
ChatGPT's — replies can get split / merged / retry-rewritten in
place, which breaks the "sentCount indexes into a stable list"
assumption that incremental mode makes. Phase 4 will revisit once
e2e evidence is in.

### Why no `pce-manual-capture` listener for Claude?

The legacy `claude.js` never wired one. Adding it now would be a
behaviour change, not a pure port. Phase 3b batch 5 can address this
as a deliberate enhancement PR.

## Risks mitigated

- **Runtime regressions in all three sites**: covered by 65 site-level
  tests + 13 runtime tests; all run in happy-dom with real DOM
  selectors.
- **Manifest misalignment**: The reduced-bundle entry was added
  explicitly so `behavior_tracker` and `text_collector` remain loaded
  on ported sites. `wxt.config.ts` comments point future maintainers
  at the split.
- **Accidental behaviour drift**: Every timing constant, regex, and
  selector was lifted byte-for-byte. The parity table above enumerates
  the guarantees and the tests that lock them in.
